# Shell 安装脚本完成后缺少旧版本清理交互

**状态**：Fixed
**优先级**：中
**创建日期**：2026-05-30

## 问题描述

`install.sh` 和 `install.ps1` 安装新版本到 `~/.peri/agent-vX.Y/` 目录后，从不清理旧版本目录。多次安装后 `~/.peri/` 下会累积大量旧版本目录（如 `agent-v1.14`、`agent-v1.15`、`agent-v1.16`...），占用磁盘空间。用户期望安装完成后能扫描旧版本、展示列表、经确认后删除。

## 症状详情

| 维度 | 当前行为 | 期望行为 |
|------|----------|----------|
| 安装后清理 | 无清理逻辑，旧版本永久保留 | 安装完成后扫描并列出旧版本 |
| 用户确认 | 不适用 | 列出旧版本（版本号 + 目录大小），用户一键确认删除 |
| 跳过清理 | 不适用 | 用户可选择跳过不清理 |

当前 `install.sh` 在第 192 行写入 `current-version.txt` 后直接结束，`install.ps1` 同理在第 184 行写完版本文件后结束，均无清理步骤。

## 涉及文件

- `scripts/install.sh` —— Unix/macOS 安装脚本（主要目标，第 223 行 `main` 结束前需加清理逻辑）
- `scripts/install.ps1` —— Windows 安装脚本（第 209 行 `Main` 结束前需加清理逻辑）

## 期望改进方向

1. 安装完成后，扫描 `${INSTALL_DIR}` 下的 `agent-v*` 目录
2. 排除当前刚安装的版本（`symlink` 指向的目录）和正在使用的版本
3. 列出所有旧版本及其目录大小
4. 提示用户是否删除，用户确认后执行 `rm -rf`，用户跳过则不删除
5. Shell 脚本中用 `read -p` 实现交互确认，PowerShell 用 `Read-Host` 实现
